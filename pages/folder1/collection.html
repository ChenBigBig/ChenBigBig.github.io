          # 集合

          # 集合的介绍

          集合和数组一样,都是可以保存一组元素.并且提供了很多操作元素相关的方法.使用更加方便
          java.util.Collection是所有集合的顶级接口
          Collection下面有多种实现类,因此我们有更多的数据结构可供选择.
          1.java.util.List线性表 是可重复的集合并且是有序的
          2.Java.util.Set不可重复集合大部分是无序的
          3.Java.util.Queue 队列 保持的是先进先出的原则
          集合只能存放引用类型的元素 并且存放的是元素的引用类型。

          # ArrayList和LinkedList区别

          ArrayList基于动态数组实现；LinkedList基于链表实现。
          对于随机index访问的get和set方法，ArrayList的速度要优于LinkedList。因为ArrayList直接通过数组下标直接找到元素；LinkedList要移动指针遍历每个元素直到找到为止。
          新增和删除元素，LinkedList的速度要优于ArrayList。因为ArrayList在新增和删除元素时，可能扩容和复制数组；LinkedList实例化对象需要时间外，只需要修改指针即可。

          # Map查找表

          Map的体现就是以键值对的形式保存数据（key和value）
          Java.util.map接口是所有Map的顶级接口，规定了Map的相关功能
          Java.util.HashMap 称为散列表,使用散列算法实现Map 当今速度是最快的
          Java.util.TreeMap 使用二叉树实现

          # HashMap实现原理

          HashMap 基于 Hash 算法实现的，我们通过 put(key,value)存储，get(key)来获取。当传入 key 时，HashMap 会根据 key. hashCode() 计算出 hash 值，根据 hash 值将 value 保存在 bucket 里。当计算出的 hash 值相同时，我们称之为 hash 冲突，HashMap 的做法是用链表和红黑树存储相同 hash 值的 value。当 hash 冲突的个数比较少时，使用链表否则使用红黑树。
